# cutPDF

> 이 소프트웨어는 본인의 귀찮은 업무를 안전하게 로컬에서 돌리기 위해 바이브 코딩을 통해 제작되었음을 밝힙니다. (심지어 리드미도)

`cutPDF`는 단일 HTML 파일로 구성된 로컬 PDF 분할 도구입니다.  
별도의 백엔드나 복잡한 프레임워크 없이, **HTML, CSS, JavaScript만으로 PDF를 나누는 웹 인터페이스**를 구현하는 것을 목표로 만들었습니다.

이 프로젝트의 핵심은 하나의 HTML 파일 안에 **구조, 스타일, 동작 로직을 모두 담은 독립형 웹 앱**이라는 점입니다.

---

## 소개

`cutPDF`는 브라우저에서 PDF를 불러오고, 원하는 페이지 범위를 입력해 여러 개의 부분 PDF로 나눌 수 있는 로컬 웹툴입니다.

서버를 두고 운영하는 서비스라기보다,  
**한 개의 HTML 파일만으로 바로 실행할 수 있는 소형 문서 유틸리티**에 가깝습니다.

---

## 어떻게 만들었는가

이 프로젝트는 하나의 HTML 파일 안에 다음 요소를 함께 담는 방식으로 만들었습니다.

- **HTML**: 업로드 영역, 파일 정보 카드, 구간 입력 리스트, 다운로드 버튼 등 전체 UI 구조
- **CSS**: 카드 중심 레이아웃, 여백, 타이포, 버튼/입력창 스타일
- **JavaScript**: PDF 업로드 처리, 페이지 범위 검증, 부분 PDF 생성, ZIP 다운로드, 동적 구간 추가/삭제

즉, `cutPDF`는 **HTML 안에 CSS와 JavaScript를 함께 내장한 self-contained 구조**로 구성되어 있습니다.

---

## 구현 방식

PDF 처리에는 `pdf-lib`를 사용했습니다.  
업로드한 원본 PDF에서 사용자가 지정한 페이지 범위만 복사해 새로운 PDF를 생성합니다.

전체 다운로드는 `JSZip`을 사용해,  
여러 개의 부분 PDF를 ZIP 파일 하나로 묶어 내려받을 수 있도록 구현했습니다.

구간 입력 UI는 고정된 폼이 아니라 JavaScript로 동적으로 생성되며,  
각 구간은 독립적으로 생성·삭제·다운로드할 수 있습니다.

---

## 왜 단일 HTML 파일인가

이 프로젝트는 의도적으로 **하나의 HTML 파일로 관리할 수 있도록** 설계했습니다.

이 방식의 장점은 다음과 같습니다.

- 구조가 단순함
- 로컬에서 바로 실행 가능함
- 유지보수가 쉬움
- 작은 유틸리티 프로젝트에 잘 맞음
- 파일 하나만으로 공유하거나 보관하기 편함

---

## 기술 스택

- HTML
- CSS
- JavaScript
- `pdf-lib`
- `JSZip`

---

## 앞으로 추가할 기능

현재는 PDF 분할 기능을 중심으로 구성되어 있지만,  
추후에는 **여러 개의 PDF를 하나로 합치는 PDF 병합 기능**도 추가할 예정입니다.

---

## 한 줄 정리

`cutPDF`는 **하나의 HTML 파일만으로 만든 로컬 PDF 분할 웹툴**이며,  
브라우저 안에서 PDF 분할과 ZIP 다운로드를 처리하는 가벼운 문서 유틸리티 프로젝트입니다.
